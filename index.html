The Semantic Grid System

Page layout for tomorrow.

Set column and gutter widths, choose the number of columns, and switch between pixels and percentages.

All without any ugly .grid_x classes in your markup. Oh, and did we mention it's responsive?

Tutorials

Fluid, but with a maximum width

A fluid layout with a max width — such as this site itself — is easily achieved by wrapping the columns in a container.

<div class="center">
   <article>...</article>
   <aside>...</aside>
</div>

div.center {
   max-width: 1200px;
   margin: 0 auto;
}
article {
   .column(9);
}
aside {
   .column(3);
}

Responsive layouts using @media queries

The Semantic Grid, with its clean separation of declarative HTML and presentational CSS, is an ideal tool for powering adaptive layouts.

article {
   .column(6);
}
aside {
   .column(6);
}

@media (max-device-width: 960px) {
   article {
      .column(12);
   }
   aside {
      .column(12);
   }
}

Getting started

Add the resources to your HTML page

Reference your LESS CSS stylesheet and the LESS.js script on your HTML page (the stylesheet must appear before the script).

<link rel="stylesheet" href="css/screen.less" type="text/less" media="screen" />
<script src="js/less-1.1.3.min.js" type="text/javascript" ></script>

Configure the CSS

Inside your stylesheet, import the grid.less file:

@import 'grid.less';

The Semantic Grid defaults to a 12 column, 960 pixel grid system. Customize the columns and gutters by overriding these three variables (after grid.less has been imported):

@column-width: 60;
@gutter-width: 20;
@columns: 12;

To use a fluid (percentage-based) grid rather than a fixed pixels, simply override one additional variable:

@total-width: 100%;

Constructing page layouts

Columns: the heart of it all

The Semantic Grid avoids presentational CSS classes in the HTML markup itself. Instead, it uses a clever parametric LESS mixin to control the layout in the stylesheet where it belongs. Below is an example of the LESS syntax, and the resulting CSS:

// This LESS rule...
article {
	.column(9);
}

// Is compiled to...
article {
	display: inline;
	float: left;
	width: 700px;
	margin: 0 10px;
}

Rows: convenient clearing

It's not necessary to insert an extra element or use .clearfix to clear floating columns. Columns can even be added directly within the body tag, without the need for a surrounding row element. Columns and nested rows all clear any floating elements within them using overflow: hidden. The body tag then clears the columns within it using the little known technique body { float: left; }.

Nested columns

Columns within columns can be a @#$% when working with percentages, but the Semantic Grid conveniently does all the calculations. There's just one catch: there must be a row surrounding the nested columns, and the nested columns need one additional parameter:

<section>
   <header>
      <h1>Title</h1>
      <nav>...</nav>
   </header>
</section>

section {
   .column(6);
	
   header {
      .row(6);
		
      h1 {
         .column(3,6);
      }
      nav {
         .column(3,6);
      }
   }
}
